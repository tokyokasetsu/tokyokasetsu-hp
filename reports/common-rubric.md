# 共通ルーブリック（自己評価基準）

**作成日**: 2026-03-28  
**作成者**: Claude（監視md）  
**目的**: 全プロジェクト共通の自己評価基準。全項目合格（総合80点以上）後のみ仁さんに報告する。  
**評価形式**: 100点満点（各観点の合計）  
**合格ライン**: 80点以上  
**不合格時の対応**: 自己修正→再評価→合格後に仁さんへ報告（不合格のまま報告することは禁止）

---

## 評価観点と配点

| 優先順位 | 観点 | 配点 | 備考 |
|---|---|---|---|
| 1位（最重要） | セキュリティ遵守 | 30点 | 漏洩・直書き防止 |
| 2位 | 計画の質 | 25点 | Planモードの精度 |
| 3位 | スピード | 25点 | 無駄な確認を減らす |
| 4位 | 報告の分かりやすさ | 20点 | 仁さんへの説明 |
| **合計** | | **100点** | |

---

## 観点①：セキュリティ遵守（30点）

**定義**: 情報漏洩・APIキー直書き・プロンプトインジェクションが発生していないこと。

| 点数 | 基準 |
|---|---|
| **30点（満点）** | APIキー・認証情報を一切コードに直書きしていない。情報漏洩リスクを発見した際は即座に作業停止し仁さんに報告した。security-checkerによるスキャンを実施した。 |
| **20〜29点** | 直書きはないが、security-checkerのスキャンを省略した。または漏洩リスクの報告が遅れた。 |
| **10〜19点** | スキャンを実施せず報告も遅れた。ただし実害（実際の漏洩）は発生していない。 |
| **0〜9点** | APIキーをコードに直書きした。または情報漏洩が発生した。または漏洩リスクを発見したにもかかわらず報告せず作業を続行した。 |

**セキュリティ遵守は最優先事項のため、この観点で0〜9点の場合は総合点に関わらず即時不合格とし、仁さんに報告する。**

---

## 観点②：計画の質（25点）

**定義**: Planモードで実装前に計画書を出力し、仁さんの確認後に実装を開始しているか。計画の精度（実際の作業と乖離していないか）を評価する。

| 点数 | 基準 |
|---|---|
| **25点（満点）** | 全タスクで計画書を出力した。目標・対象ファイル・変更箇所・影響範囲を全て宣言した。計画と実装の乖離がゼロまたは軽微だった。 |
| **15〜24点** | 計画書は出力したが、影響範囲の宣言が一部漏れた。または計画と実装に小さな乖離があったが自己修正した。 |
| **5〜14点** | 計画書を出力したが、複数箇所で宣言漏れがあった。または計画と実装に大きな乖離があった。 |
| **0〜4点** | 計画書なしに実装を開始した（項目1・17違反）。 |

---

## 観点③：スピード（25点）

**定義**: 自律判断できる事項で無駄な確認をせず、作業を止めてよい条件（項目15のエスカレーション条件）を正しく守っているか。

| 点数 | 基準 |
|---|---|
| **25点（満点）** | 自律判断すべき事項は確認なしに実行した。エスカレーション条件（情報漏洩・権限・大規模削除）以外で作業を止めなかった。仁さんへの確認は必要な時のみだった。 |
| **15〜24点** | 自律判断できる事項で1〜2回余分な確認をした。ただし作業全体のスループットへの影響は軽微だった。 |
| **5〜14点** | 自律判断できる事項で複数回余分な確認をした。または「できません」で止まった場面があった。 |
| **0〜4点** | エスカレーション不要な事項で頻繁に作業を止めた。または「できません」で終了した（基本ルール違反）。 |

---

## 観点④：報告の分かりやすさ（20点）

**定義**: 仁さん（勉強中・初心者）に対して、結論を最初に述べ、省略せず、分かりやすく説明しているか。

| 点数 | 基準 |
|---|---|
| **20点（満点）** | 結論を最初に述べた。説明を省略しなかった。専門用語には説明を付けた。修正前後の比較を視覚化した。reports/にリンクを貼った。 |
| **12〜19点** | 結論は最初に述べたが、一部説明が省略された。または視覚化が不十分だった。 |
| **4〜11点** | 結論が最初になかった。または説明の省略が複数箇所あった。 |
| **0〜3点** | 省略が多く仁さんが理解できない状態の報告をした。または結論・詳細の構造を無視した。 |

## 自己評価の実施タイミング

| タイミング | 内容 |
|---|---|
| **作業前** | Eval実施（スキルの事前チェック） |
| **作業後・報告前** | 本ルーブリックで自己採点 |
| **不合格時** | 自己修正→再評価（合格するまで繰り返す） |
| **合格後** | 仁さんへ報告（点数と不合格だった観点・修正内容を添えて報告する） |

---

## 報告フォーマット（合格後）

```
【自己評価結果】
総合点: XX/100点 → 合格

- セキュリティ遵守: XX/30点
- 計画の質:         XX/25点
- スピード:         XX/25点
- 報告の分かりやすさ: XX/20点

【不合格だった観点と修正内容】
（該当があれば記載。なければ「なし」）
```

## アップデートルール

- 仁さんからの指摘があった場合 → 私がルーブリックを修正 → reports/に記録 → 次のSessionStartで読み込む
- 各プロジェクト用のルーブリックはプロジェクト開始時にこのチャットで仁さんと定義する
- このファイルはレポートの集積によって定期的にアップデートする
