## 問題1: ヘッダー文字被り（crest-area の padding-top）

### 何が起きていたか

fixedヘッダー（高さ80px）の直下にある `.crest-area` が `padding-top:80px` を使用していた。`padding-top` は要素の**内側**に余白を作るため、ヘッダーが紋章エリアのコンテンツに重なって文字・UIが被っていた。

### 修正前のコード

```css
/* デスクトップ（L169） */
.crest-area{
  position:relative;width:100%;height:35svh;
  overflow:hidden;display:flex;align-items:center;justify-content:center;
  padding-top:80px;    /* ← NG: 内側余白のためヘッダーと被る */
  background:#ffffff;
  cursor:default;
}

/* レスポンシブ ≤1024px（L429） */
.crest-area{padding-top:64px}   /* ← NG: 同上 */
```

### 修正後のコード

```css
/* デスクトップ */
.crest-area{
  ...
  margin-top:80px;     /* ← OK: 外側余白でヘッダー分を確保 */
  ...
}

/* レスポンシブ ≤1024px */
.crest-area{margin-top:64px}    /* ← OK */
```

### なぜ margin-top を選んだか

- `padding-top` は要素の**ボックス内側**に余白を作る。`overflow:hidden` が設定されている `.crest-area` では、padding領域もボックス内部に含まれるため、fixedヘッダー（z-index上位）がpadding領域ごと覆い隠す。
- `margin-top` は要素の**ボックス外側**に余白を作る。要素自体がヘッダーの下に押し下げられるため、コンテンツがヘッダーに隠れない。
- 04-hp-spec.md に「fixed ヘッダー下のコンテンツは `margin-top` で余白を取る（`padding-top` 禁止）」と明記されている。

### なぜこの問題が残っていたか

status.md には「padding-top → margin-top ✅ .crest-area 2箇所修正」と記録されていたが、**実際のコードには padding-top が残っていた**。status.md の記録と実コードの乖離が発生していた。

## 問題2: showStd()/showSpc() 関数が未定義

### 何が起きていたか

ローカルナビのクリックイベント（L877-878）で `showStd()` / `showSpc()` を呼び出していたが、**関数本体がファイル内のどこにも定義されていなかった**。紋章ゾーンのhover/clickイベントリスナーも存在しなかった。結果、紋章UIをクリックしても何も起こらず、通常足場/特殊足場パネルが一切表示されなかった。

### 修正前の状態

```javascript
/* Local nav click: std/spc triggers panel */
lLinks.forEach(function(a){
  a.addEventListener('click',function(e){
    var s=a.dataset.section;
    if(s==='std'){e.preventDefault();showStd();}   // ← ReferenceError: showStd is not defined
    else if(s==='spc'){e.preventDefault();showSpc();} // ← 同上
  });
});

// ゾーン（.zone--std / .zone--spc）のイベントリスナー → 存在しない
```

### 修正後のコード

```javascript
/* === showStd / showSpc : 紋章ゾーン切り替え === */
var crestArea=document.getElementById('CA');
var contentPanels=document.getElementById('contentPanels');
var panelStd=document.getElementById('panelStd');
var panelSpc=document.getElementById('panelSpc');
var zoneStd=document.getElementById('zoneStd');
var zoneSpc=document.getElementById('zoneSpc');

function showStd(){
  crestArea.classList.add('picked','show-std');
  crestArea.classList.remove('show-spc');
  contentPanels.classList.add('open','bg-std');
  contentPanels.classList.remove('bg-spc');
  panelStd.classList.add('on');
  panelSpc.classList.remove('on');
  setTimeout(function(){
    panelStd.querySelectorAll('.method__text').forEach(function(el){el.classList.add('in')});
  },200);
  contentPanels.scrollIntoView({behavior:'smooth',block:'start'});
}
function showSpc(){
  crestArea.classList.add('picked','show-spc');
  crestArea.classList.remove('show-std');
  contentPanels.classList.add('open','bg-spc');
  contentPanels.classList.remove('bg-std');
  panelSpc.classList.add('on');
  panelStd.classList.remove('on');
  setTimeout(function(){
    panelSpc.querySelectorAll('.method__text').forEach(function(el){el.classList.add('in')});
  },200);
  contentPanels.scrollIntoView({behavior:'smooth',block:'start'});
}

/* ゾーン hover/click イベント */
zoneStd.addEventListener('mouseenter',function(){
  crestArea.classList.add('show-std');
  crestArea.classList.remove('show-spc');
});
zoneSpc.addEventListener('mouseenter',function(){
  crestArea.classList.add('show-spc');
  crestArea.classList.remove('show-std');
});
zoneStd.addEventListener('mouseleave',function(){
  if(!crestArea.classList.contains('picked')){
    crestArea.classList.remove('show-std');
  }
});
zoneSpc.addEventListener('mouseleave',function(){
  if(!crestArea.classList.contains('picked')){
    crestArea.classList.remove('show-spc');
  }
});
zoneStd.addEventListener('click',function(){showStd()});
zoneSpc.addEventListener('click',function(){showSpc()});
```

### なぜこの実装にしたか

1. **CSSクラス駆動方式**: 既存CSSに `.crest-area.show-std`、`.content-panels.open`、`.panel.on` 等のスタイルが全て定義済みだった。JSで直接スタイルを書くのではなく、クラスの付け外しでCSSに委譲する設計を踏襲した。

2. **hover と click の分離**: hover時は羽根のハイライトのみ（`show-std`/`show-spc`）、click時にパネル展開（`open`/`on`/`picked`）と段階的に動作させた。`picked` クラスで「クリック確定済み」を管理し、mouseleave時にhoverエフェクトが消えないようにした。

3. **method__text の .in クラス**: パネル展開後に200ms遅延で `.method__text` に `.in` を付与。既存CSSの `opacity:0;transform:translateY(60px)` → `opacity:1;transform:translateY(0)` トランジションを発火させ、テキストが下からスライドインするアニメーションを実現した。

4. **scrollIntoView**: パネル展開後に自動スクロールすることで、ユーザーが手動でスクロールする手間を省いた。

## 再発防止策

### padding-top 問題

- **04-hp-spec.md のルール確認を徹底する**: fixedヘッダー下では `margin-top` のみ使用。`padding-top` は禁止。
- **status.md と実コードの整合性確認**: status.mdに「修正済み」と記録する前に、実コードで `grep` して確認する。記録と実態の乖離は信頼性を損なう。

### 関数未定義問題

- **関数呼び出しを書いたら、同一セッション内で必ず関数定義も書く**: 呼び出し側だけ実装して定義を忘れるのは典型的な実装漏れ。
- **ブラウザのコンソールで動作確認する習慣**: `ReferenceError` は開発者ツールのコンソールで即座に検出できる。
- **status.md に「実装済み」と書く前にブラウザ確認**: UIの動作確認なしに完了報告しない。
