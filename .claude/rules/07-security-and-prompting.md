# セキュリティ意識・プロンプトキーワード

## セキュリティ意識

### コード理解の原則
- コードを理解せずに動かすことのリスクを常に意識する
- 「何をしているか」は最低限把握する習慣を持つ

### SaaS開発でのセキュリティ
- SaaS開発ではセキュリティレビューのサブエージェントを必ず組み込む
- コード生成後に自動でセキュリティチェックを走らせる設計にする

## プロンプトキーワード習慣

特定のキーワードをプロンプトに含めるだけで出力傾向が変わる:

| キーワード | 効果 |
|---|---|
| `interactive` | インタラクティブな要素（操作可能UI等）を生成 |
| `visualize` | 視覚的な出力（チャート・図・グラフィカル表現）になる |
| `ultra think` | 最大レベルの思考量で回答 |
| `think hard` | 深い分析・多角的な検討を促す |

意図した出力形式に近づけるための簡易テクニック。

## プロンプト技術（高品質出力のための4原則）

### Persistence（粘り強さ）
- 不確実でも停止せず、最後まで続行させる
- 途中で「わかりません」と止まらず、仮説を立てて進む

### Code Editing Rules（コード編集ルール）
- 使用技術・命名規則・コーディング規約を事前に定義して渡す
- 作業開始前にルールを読み込ませることで一貫性を担保

### Tool Prims（ツール呼び出しプライム）
- ツール呼び出し前に、目標を言い換えてナレーションさせる
- 意図のズレを防ぎ、正確なツール使用を促す

### Self Reflection（自己反省）
- 自分でルーブリック（評価基準）を作成させる
- 最高点になるまで自らやり直させる仕組み
