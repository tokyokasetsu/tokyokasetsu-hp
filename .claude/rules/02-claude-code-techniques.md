# Claude Code 操作テクニック

## 基本操作

- **Planモード**: Shift+Tab を2回押して切替。実装前に設計方針を立てるときに使用
- **/rewind**: 直前の操作を巻き戻す。ミスした変更を即座に取り消せる
- **割り込み入力**: Claude の応答中でもテキストを入力可能。方向修正や追加指示に使う
- **/context コマンド**: 会話にコンテキスト（ファイル・URL等）を追加する

## .claude/rules/ 構成

- `.claude/rules/` にルールファイルを配置すると、Claude Code が自動で読み込む
- CLAUDE.md が肥大化したら、ここにテーマ別に分散する
- ファイル名の番号プレフィックスで読み込み順を制御（例: `01-`, `02-`）

## コンテキストエンジニアリング

- Claude に渡すコンテキスト（指示・ファイル・ルール）の質が出力品質を決める
- 必要な情報を過不足なく渡し、ノイズを減らすことが重要
- CLAUDE.md と .claude/rules/ の使い分けがその実践

## CLAUDE.md 運用の5コツ

1. **Less is More** — 短く的確に。長文は読み飛ばされる
2. **必要な時だけ出す** — 常時読み込みが不要なルールは .claude/rules/ へ分離
3. **How 重視** — 「何をするか」より「どうやるか」を具体的に書く
4. **検証手段を明記** — ルールの遵守を確認できるチェック方法を添える
5. **痛みから育てる** — 実際のミスや問題から逆算してルールを追加する
