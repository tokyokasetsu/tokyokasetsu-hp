# 開発手法・ツール連携ナレッジ

## バイブコーディング手法

### UIガチャ
- 同じプロンプトで複数回生成し、最も良いUI出力を選ぶ手法
- デザインの探索・比較に有効

### パラレルバイブコーディング
- 複数のAIエージェントを並列で走らせ、同時に異なるタスクを進める手法
- 独立した作業を同時並行で処理し、開発速度を大幅に向上

## ツール・MCP

### skills.sh
- Claude Code のスキル（カスタムスラッシュコマンド）を定義するシェルスクリプト

### ui-ux-pro-max
- UI/UXデザインに特化したプロンプトまたはスキルセット

### Google Stitch MCP
- Google Stitch をMCPサーバーとして接続し、Claude Code から直接操作する仕組み

### Figma MCP 実装
- Figma をMCPサーバーとして接続し、デザインデータを Claude Code に取り込む
- デザイン→実装の変換精度を向上させる

### Claude Dispatch
- 複数の Claude エージェントを管理・振り分けする仕組み
- タスクの種類に応じて適切なエージェントに自動ルーティング

## SaaS 開発フロー

推奨ワークフロー:

```
Claude Code（設計・コア実装）
  → Bolt.new（プロトタイプ・UI生成）
    → GitHub（ソース管理）
      → Cursor（詳細実装・デバッグ）
        → Vercel（デプロイ・公開）
```

各ツールの得意領域を活かしたリレー開発。

## 要件定義ワークフロー

### 録音→要件定義
- クライアントとの会話を録音 → 文字起こし → Claude で要件定義書に整形
- ヒアリング漏れを防ぎ、認識齟齬を最小化

### 要求定義 vs 要件定義

| | 要求定義 | 要件定義 |
|---|---|---|
| 誰が | クライアント・ユーザー | 開発チーム |
| 内容 | 「こうしたい」（願望・目的） | 「こう作る」（仕様・制約） |
| 場面 | チャットでの対話がまさにこれ | 要求を受けて技術的に整理 |

**このチャット（Claude Code との対話）は要求定義の場**。ユーザーの「こうしたい」を受け取り、技術的な要件に変換するのが Claude の役割。
