# 開発手法・ツール連携ナレッジ

## バイブコーディング手法

### UIガチャ
- 同じプロンプトで複数回生成し、最も良いUI出力を選ぶ手法
- デザインの探索・比較に有効

### パラレルバイブコーディング
- 複数のAIエージェントを並列で走らせ、同時に異なるタスクを進める手法
- 独立した作業を同時並行で処理し、開発速度を大幅に向上

## ツール・MCP

### skills.sh
- Claude Code のスキル（カスタムスラッシュコマンド）を定義するシェルスクリプト

### ui-ux-pro-max
- UI/UXデザインに特化したプロンプトまたはスキルセット

### Google Stitch MCP
- Google Stitch をMCPサーバーとして接続し、Claude Code から直接操作する仕組み

### Figma MCP 実装
- Figma をMCPサーバーとして接続し、デザインデータを Claude Code に取り込む
- デザイン→実装の変換精度を向上させる

### Claude Dispatch
- 複数の Claude エージェントを管理・振り分けする仕組み
- タスクの種類に応じて適切なエージェントに自動ルーティング

### Context 7 MCP
- プロンプトに「use context 7」と入れるだけで最新ライブラリドキュメントを自動参照
- 古いバージョンのAPIを使ってしまう問題を解決

## SaaS 開発フロー

推奨ワークフロー:

```
Claude Code（設計・コア実装）
  → Bolt.new（プロトタイプ・UI生成）
    → GitHub（ソース管理）
      → Cursor（詳細実装・デバッグ）
        → Vercel（デプロイ・公開）
```

各ツールの得意領域を活かしたリレー開発。

## AI分業フロー

### Gemini × Claude Code 分業
- **Gemini**: 設計・要件整理（トークン単価が安い）
- **Claude Code**: 実装・コード生成
- 設計フェーズのトークン消費を節約しつつ、実装品質を最大化

### Cowork スケジュールタスク
- `/schedule` コマンドで定期実行タスクを設定
- 自動化可能な繰り返し作業に活用

### Claude in PowerPoint
- テキスト・グラフを一括更新
- デザイン崩れなしでプレゼン資料を自動生成・更新

## Cowork vs Claude Code 使い分け

| ユースケース | 推奨ツール |
|---|---|
| API連携が必要 | Claude Code |
| MCP対応アプリ・PC作業自動化 | Cowork |
| 非エンジニアの日常業務 | Cowork（壁にぶつかったらClaude Codeへ） |
