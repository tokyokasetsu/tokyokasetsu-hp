# スキル設計ナレッジ

## 公式フォルダー構造

- `skill.md` が必須（エントリポイント）
- サブフォルダ: `scripts/` / `references/` / `assets/`

## YAMLフロントマター仕様

- 必須項目: 名前・説明・トリガーフレーズ
- 説明は **1024文字以内**
- **XMLタグ禁止**（フロントマター内で使わない）

## スキルの2分類

| 種類 | 特徴 | 寿命 |
|---|---|---|
| 能力引き上げ型 | モデルの弱点を補うスキル | モデル進化で不要化する |
| プロセス組み込み型 | ワークフローを定義するスキル | 長持ちする（推奨） |

## 品質管理（Eval）

- **チェックリスト**で品質劣化を検知する
- **ベンチマーク**: スキルあり / なしの出力を数値で比較
- 定期的にEvalを回して品質維持を確認

## Description 自動最適化

- スキルが発動しない問題は、Description の記述が原因であることが多い
- トリガーフレーズとDescriptionを最適化して解決する

## サブエージェント設計

### 定義の原則
- **ロールではなくタスクで定義する**
  - NG: 「フロントエンド開発者」（役割）
  - OK: 「コードを最適化する」（タスク）
- タスクベースの方が出力が具体的で再現性が高い

### 操作
- `/agents` コマンドでエージェントを作成
- `@エージェント名` で呼び出し

## 運用上の注意

- スキルが増えすぎると**スパゲッティ化**する → 定期的に整理・削除が必要
- **agents.MD**（毎回全部読み込む方式）の方が精度が高いケースもある
- スキル分割 vs 一括読み込みは、プロジェクト規模と精度要件で判断
